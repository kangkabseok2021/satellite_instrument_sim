## 1. Scope

This SRS defines the requirements for the Satellite Instrument Simulator — a software suite
simulating a space-borne atmospheric lidar (EarthCARE ATLID class) for development and
test purposes.

## 2. Applicable Documents

- ICD-001: Interface Control Document (binary frame layout)
- ECSS-E-ST-40C: Software Engineering Standard

## 3. System Overview

The system consists of two services:
- **Simulator (C++):** generates telemetry frames at 1 kHz over a POSIX named pipe
- **Processor (Python):** ingests frames, applies signal processing, exposes REST metrics

## 4. Functional Requirements

| ID | Requirement | Test |
|---|---|---|
| FR-001 | Simulator shall emit telemetry frames at 1000 Hz ± 50 Hz | test_atmospheric.cpp::BackscatterDecaysWithRange |
| FR-002 | Each frame shall be 16 bytes, little-endian, per ICD-001 | test_frame.cpp::Crc8RoundTrip |
| FR-003 | Simulator shall compute backscatter using Beer-Lambert law | test_atmospheric.cpp::BackscatterDecaysWithRange |
| FR-004 | Simulator shall add Gaussian shot noise with configurable sigma | test_atmospheric.cpp::SnrDecreasesWithRange |
| FR-005 | Processor shall detect and reject frames with invalid CRC-8 | test_ingestor.py::test_bad_crc_raises |
| FR-006 | Processor shall apply 4th-order Butterworth LPF (100 Hz cutoff) | test_processor.py::test_filtered_signal_reduces_rms_noise |
| FR-007 | Processor shall estimate CO2 column via Beer-Lambert inversion | test_processor.py::test_co2_retrieval_positive |
| FR-008 | Processor shall expose GET /api/metrics (rolling 10-frame average) | test_api.py::test_metrics_schema |
| FR-009 | Processor shall expose GET /api/health returning {"status":"ok"} | test_api.py::test_health_returns_200 |
| FR-010 | Processor shall expose GET /api/frames/{n} returning last n frames | test_api.py::test_frames_endpoint |
| FR-011 | Simulator shall set solar_flag=1 when SIM_SOLAR env var is set | test_atmospheric.cpp::SolarFlagSetWhenContaminated |
| FR-012 | Both services shall shut down gracefully on SIGTERM | src/main.cpp (SIGTERM handler) |

## 5. Non-Functional Requirements

| ID | Requirement |
|---|---|
| NFR-001 | Simulator CPU utilisation < 20% on a single core at 1 kHz |
| NFR-002 | Simulator memory footprint < 64 MB |
| NFR-003 | Named pipe write latency < 5 ms P99 per frame |
| NFR-004 | Docker runtime images < 100 MB each |
| NFR-005 | pytest branch coverage >= 90% on processor/ |

## 6. Verification Matrix

All FR-* items are verified by the test cases listed in Section 4.
NFR-001 and NFR-002 verified by runtime measurement under `docker stats`.
NFR-004 verified by `docker images`.
