## 1. Interface Overview

The C++ Simulator (writer) and Python Processor (reader) communicate via a POSIX
named pipe. The Simulator writes binary telemetry frames at 1 kHz. The Processor reads
and processes each frame sequentially.

**Transport:** POSIX FIFO (`mkfifo`, `O_WRONLY` / `O_RDONLY`)
**Byte order:** Little-endian
**Frame size:** Fixed 16 bytes

## 3. CRC-8 Specification

- **Polynomial:** 0x31 (x‚Å∏ + x‚Åµ + x‚Å¥ + 1)
- **Initial value:** 0xFF
- **Reflection:** none (MSB-first)
- **XOR output:** none
- **Coverage:** bytes 0‚Äì14 (15 bytes; byte 15 is the CRC field itself)

**Test vector:** `CRC8([0x00, 0x00, 0x00], init=0xFF) = 0xF7`

**Algorithm (Python):**
```python
def crc8(data: bytes, init: int = 0xFF) -> int:
    crc = init
    for byte in data:
        crc ^= byte
        for _ in range(8):
            crc = ((crc << 1) & 0xFF) ^ 0x31 if (crc & 0x80) else (crc << 1) & 0xFF
    return crc
```

## 4. Error Handling

| Condition | Processor action |
|---|---|
| `len(frame) != 16` | Raise `CorruptFrameError`, skip frame |
| CRC mismatch | Raise `CorruptFrameError`, skip frame, log warning |
| `channel_id` not in {0,1,2,3} | Raise Pydantic `ValidationError`, skip frame |
| `altitude_m` > 65535 | Raise Pydantic `ValidationError`, skip frame |
| Pipe closed (EOF) | Exit `ingest_loop` gracefully |

---

## 5. Change Control

Any change to the frame layout or CRC algorithm requires:
1. Increment ICD-001 version number
2. Update both `TelemetryFrame.h` (C++) and `frame_ingestor.py` (Python)
3. Update all affected test vectors
4. Create a PR referencing the ICD change
